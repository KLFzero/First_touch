# Unity 为主轴的 12 周 Game Jam 准备日程表（可打印版）

说明：
- 你的时间安排：周一—周五每天 1–2 小时，周六 1 小时回顾，周日 3–4 小时深做。每周总计约 9–15 小时。
- 本日程表把每周任务细化为“最小可交付单元（MMF）”，便于每天完成并保持持续产出与记录。
- 每周末（周六）做 1 小时回顾并写 devlog；周日做深度实现与合并（build、录屏、提交等）。

使用说明（打印建议）
- 建议每周打印一页，或将 12 周缩成 3 页（每页 4 周）便于粘贴墙面。  
- 每天旁边预留打勾栏（□）用于打卡。  
- 每周回顾可直接填写到 docs/devlog.md 或打印的回顾表上。

## 第 1 周 — 环境与第一个可运行项目（Roll-a-ball）
目标：安装工具并完成 Unity 官方入门，把项目上传到 GitHub。
- 周一（1–1.5h）
  - 安装 Unity Hub + LTS 版本，安装 Visual Studio/IDE
  - □
- 周二（1–1.5h）
  - 安装 Git，注册 GitHub，建仓库，添加 Unity .gitignore & LICENSE
  - □
- 周三（1–2h）
  - 在 Unity 中新建项目，熟悉 Scene/Hierarchy/Inspector/Project 面板
  - □
- 周四（1–2h）
  - 跟随 Roll-a-ball 教程：放置物体、添加 Rigidbody
  - □
- 周五（1h）
  - 实现玩家移动脚本（基础 Move），本地测试
  - □
- 周六（1h 回顾）
  - 把遇到的问题写入 docs/devlog.md，列出周日要完成的剩余项
  - □
- 周日（3–4h 深做）
  - 完成教程（计分、拾取物），提交到 dev 分支，写 README，截图上传 media
  - □

交付物：GitHub 上有初始项目、README、至少一次 commit/PR
